# ZSGate 功能矩阵（开源版 vs 商业版）

## 定位
- 开源版：解决团队级通用痛点，最大化采用与传播。
- 商业版：解决组织级治理、合规和规模化运维。

## 能力对照

| 能力 | 开源版（zsgate） | 商业版（zsgate-enterprise） |
|---|---|---|
| 多 Vendor 网关（OpenAI/Claude/Gemini/vLLM） | 支持 | 支持 |
| OpenAI 兼容接口 | 支持 | 支持 |
| 模型别名 + 权重路由 + 故障切换 | 支持 | 支持 |
| 基础 Vendor 管理后台 API | 支持 | 支持 |
| 基础用量统计与成本估算 | 支持 | 支持 |
| 团队统计（按成员/项目/部门） | 支持 | 支持 |
| 基础审计追踪与 Trace 检索 | 支持 | 支持 |
| MCP 基础集成 | 支持 | 支持 |
| 单集群私有化部署 | 支持 | 支持 |
| OIDC/SSO 企业连接器 | 不支持 | 支持 |
| 高级 RBAC 与权限包 | 不支持 | 支持 |
| 审批流与审批策略引擎 | 不支持 | 支持 |
| 合规报表模板与治理导出 | 不支持 | 支持 |
| 多集群/多地域控制面运维 | 不支持 | 支持 |
| 企业安全集成（KMS/HSM 策略包） | 不支持 | 支持 |
| SLA 与企业支持工具包 | 不支持 | 支持 |

## 边界原则
1. 不把“基础可用能力”从开源版挪到商业版。
2. 开源版必须能满足真实团队的日常使用。
3. 商业版聚焦治理深度、合规深度与规模化运维。
4. 新能力按“团队通用需求（开源）/组织治理需求（商业）”归类。

## 为什么这个边界合理
- 开源版持续有吸引力，利于增长与品牌扩散。
- 商业版保留清晰、可预算化的高价值购买点。
- 研发可并行推进两条产品线，减少定位混乱。
